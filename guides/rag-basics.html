⬅ חזרה למדריכים

מדריך מקיף: בניית RAG מקצה לקצה

מה זה RAG ולמה צריך אותו?

RAG מחבר מודל שפה לידע אמיתי מתוך המסמכים שלך. במקום "להמציא", המודל נשען על מקורות מאוחזרים ומחזיר תשובה עם ציטוטים.

ארכיטקטורה מעשית

  1. איסוף מסמכים (PDF/Notion/Docs)
  2. ניקוי טקסט וחלוקה ל-chunks
  3. יצירת Embeddings לכל chunk
  4. שמירה ב-Vector DB
  5. בעת שאילתה: Retrieval → Generation

דוגמה מלאה (Pseudo-code)

# 1) טעינת מסמכים
chunks = split_documents(docs, chunk_size=500, overlap=80)

# 2) embedding + index
vectors = embed(chunks)
index = vector_db.upsert(vectors, metadata=source_info)

# 3) שאילתה
q_vec = embed([user_query])
results = vector_db.search(q_vec, top_k=5)

# 4) יצירת תשובה מבוססת מקורות
context = join(results)
answer = llm.generate(
  prompt=f"ענה רק לפי ההקשר הבא:\n{context}\nשאלה:{user_query}\nציין מקורות"
)
return answer

בחירות תכנון חשובות

  • Chunk size: 350–800 טוקנים.
  • Overlap: 50–120 כדי לשמור רצף משמעות.
  • Top-k: מתחילים ב-4–6 ומכיילים לפי דיוק.
  • Filters: תאריך/מחלקה/שפה להגברת רלוונטיות.

איך מודדים איכות?

  • Retrieval quality: האם המסמכים שאוחזרו באמת רלוונטיים.
  • Faithfulness: האם התשובה נאמנה למקור.
  • Citation coverage: האם יש מקור לכל טענה.
  • Latency + Cost: זמן ועלות לכל שאילתה.

תקלות נפוצות + פתרון

  • תשובות כלליות מדי → חזק הוראה: "ענה רק מהמקורות".
  • החמצת מסמך חשוב → הגדל top-k + hybrid search.
  • כפילויות בתשובה → בצע dedup ל-chunks שחזרו.

תרגול עצמי

משימה: בנה RAG קטן ל-20 מסמכי נהלים.

יעדים: תשובה עם 2 מקורות לפחות וזמן תגובה < 4 שניות.

פתרון מוצע 
- Chunk size: 550, overlap: 80
- Embedding model איכותי (multilingual אם יש עברית)
- top-k = 5
- Prompt קשיח: "אל תענה ללא מקור."
- החזרת output בפורמט: תשובה + רשימת מקורות + confidence
