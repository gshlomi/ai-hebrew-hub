מדריך מקיף: בניית RAG בסיסי

מה זה RAG?

Retrieval-Augmented Generation: אחזור מידע רלוונטי ממקורות שלך, ואז יצירת תשובה שמבוססת על המידע שאוחזר.

ארכיטקטורה בסיסית (5 שלבים)

  1. איסוף מסמכים
  2. Chunking
  3. Embeddings + Vector DB
  4. Retrieval
  5. Generation עם ציטוטים

Pipeline מומלץ

documents -> chunks -> embeddings -> vector index
user query -> embedding -> top-k retrieval -> LLM answer + sources

החלטות חשובות

  • Chunk size: בדרך כלל 300–800 טוקנים.
  • Top-k: לרוב 3–8 מסמכים.
  • Hybrid search: וקטורי + מילולי כשצריך דיוק טרמינולוגי.
  • Always cite sources: חובה להציג מקור בכל תשובה.

מדדי איכות

  • Precision@k ו-Recall@k לאחזור.
  • Faithfulness: האם התשובה נאמנה למקור.
  • Latency: זמן תגובה כולל.

בעיות נפוצות ופתרונות

  • Hallucination → להכריח תשובה “רק מהמקור”.
  • Missing context → להגדיל top-k או chunk overlap.
  • Duplication → dedup על chunks לפני generation.
