מדריך מקיף: אוטומציה עם Agents

רמות קושי במדריך

  • מתחיל: תכנון זרימת Agent בסיסית.
  • בינוני: שימוש בכלים, ולידציה ו-guardrails.
  • מתקדם: KPI, ניטור, ושילוב תהליכים ארגוניים מרובי שלבים.

מה זה Agent בפועל?

Agent הוא תהליך שמקבל יעד, מתכנן צעדים, משתמש בכלים (API/DB/קבצים), בודק תוצאה, ומבצע תיקון אם צריך.

מתי להשתמש בסוכן?

  • כאשר יש רצף של כמה פעולות ולא פעולה בודדת.
  • כאשר צריך לקשר בין כמה מקורות מידע.
  • כאשר ערך עסקי נובע מאוטומציה חוזרת.

תבנית Agent אמינה

1) Goal Definition
2) Plan (max N steps)
3) Execute tools
4) Validate output
5) Human approval (if external action)
6) Final report + logs

דוגמה מלאה: "סיכום חדשות AI יומי"

Goal: לסכם חדשות AI יומיות לצוות הנהלה

Plan:
- שלב 1: למשוך כותרות מ-5 מקורות אמינים
- שלב 2: לסנן כפילויות ו-clickbait
- שלב 3: להפיק 5 תובנות עסקיות
- שלב 4: לצרף קישורי מקור לכל תובנה
- שלב 5: להכין הודעת סיכום ב-WhatsApp/Email לאישור אנושי

Validation:
- לפחות מקור אחד לכל פריט
- ללא טענה ללא מקור
- אורך סיכום עד 250 מילים

Guardrails חובה

  • אישור אנושי לפעולות חיצוניות (שליחה, פרסום, מחיקה).
  • הגבלת כלים ורשאות לפי משימה.
  • Timeout לכל פעולה ו-Retry מוגבל.
  • Audit log לכל צעד (מה בוצע, מתי, ולמה).

תבנית מדידה (KPI)

  • Success Rate: כמה משימות הושלמו נכון.
  • Override Rate: כמה פעמים אדם תיקן את הסוכן.
  • Latency: זמן ביצוע מקצה לקצה.
  • Cost per Task: עלות לכל משימה.

תרגול עצמי

משימה: תכנן Agent שמטפל בלידים נכנסים (מייל/טופס) ומעדכן CRM.

דרישות: סיווג ליד, העשרה קצרה ממקור חיצוני, וטיוטת תשובה מותאמת.

פתרון מוצע 
Flow מוצע:
1) Parse lead message
2) Classify intent (demo/pricing/support)
3) Enrich with company info (safe source)
4) Create CRM record
5) Draft response template
6) Ask human approval before send

Safety:
- No outbound send without approval
- Strip sensitive fields from logs
